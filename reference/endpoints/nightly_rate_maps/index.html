Overview

This resource represents Nightly Rate Map - a resource defining nightly rates and minimum stays in reference to start_date for the consecutive 1096. Itâ€™s used as a basic block for defining nightly rates - you can either use Seasons, Periods with Rentals base_rate or Nightly Rate Map

Managing Nightly Rate Maps requires external rates management on the Rental level (by setting nightly_rates_managed_externally to true).

For Rentals with internal rates management (nightly_rates_managed_externally is false) Nightly Rate Maps are regenerated everyday at 00:00 UTC and start_date always points to the previous day from current day in UTC timezone. For rentals with external rates management (nightly_rates_managed_externally is true) our system will neither refresh the map nor update start_date as this responsibility is passed onto the application managing the rates.

Update requirement

Rental attribute nightly_rates_managed_externally must be set to true, before you can write to this endpoint.

Partial update

This endpoint supports partial updates, meaning you can push only subset that changes with corresponding start_date.

Each map maximum length follows the same restrictions as start_date. Please note that only 1096 nights of data can be sent when the start_date is yesterday UTC. Therefore if sending start_date as today UTC, you can only set 1095 nights.
